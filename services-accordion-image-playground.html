Variation

Expand Reveal

Image stays completely hidden in the collapsed row. When the accordion opens, the image slides in from the right with an accent-colored wipe overlay — cinematic and clean.

Pattern Notes

How It Works

The collapsed row is pure typography — number, title, tagline, chevron. On expand, the body splits into two columns: description+tags on the left, image on the right. The image enters with a translateX(30px) slide and an accent-colored overlay that wipes away via scaleX.

View Transition Wiring

  • The <img> inside the expanded panel gets transition:name={`service-hero-${slug}`}
  • Since the image is only visible when expanded, the user must open the accordion before clicking "View Service" — this naturally ensures the transition source is visible
  • The 16:10 aspect ratio maps closely to the full-bleed hero on the detail page

Trade-offs

  • Pro: Cleanest collapsed state — identical to current design
  • Pro: Dramatic reveal moment creates anticipation
  • Con: View Transition only works if accordion is open (image must be visible)
  • Con: No visual preview without interaction

Mobile

Stacks to single column. Image appears below description. Wipe animation still works but simpler fadeIn may be better on mobile.

Persistent Thumbnail

A small desaturated thumbnail lives permanently in each row between the number and title. It colorizes on hover and grows into a larger preview when the accordion expands.

Pattern Notes

How It Works

Each row gets a 100×64px thumbnail between the number and title. In collapsed state it's desaturated at 50% opacity — a subtle hint. On hover it colorizes. On expand, a larger version appears in the body panel alongside the description.

View Transition Wiring

  • The row thumbnail gets transition:name={`service-hero-${slug}`}
  • Since it's always visible, the View Transition works from ANY state — user doesn't need to expand first
  • The browser morphs the small thumbnail → full hero in one fluid animation
  • Best option for reliable View Transitions

Trade-offs

  • Pro: View Transition always available — no expand needed
  • Pro: Adds visual richness to the list without expanding
  • Con: Adds visual weight to every row — less typographic purity
  • Con: 5 images loading on page load (lazy-load with loading="lazy")

Mobile

Thumbnail shrinks to 60×40px. Still readable. Consider hiding on very small screens (<480px).

Background Peek

A full-bleed background image hides behind each row at 0% opacity. On hover it fades to 15%, on expand to 20% — creating an atmospheric preview. A smaller framed image appears in the expanded body.

Pattern Notes

How It Works

Each accordion item has a full-width background image (CSS positioned, covers the item). A gradient overlay from left ensures text remains readable. On hover: image fades to ~15% opacity. On expand: ~20% with a small framed image in the body for View Transition.

View Transition Wiring

  • The small framed image in the expanded body gets transition:name
  • The background image is CSS-positioned so it can't be a transition source — the framed <img> is used instead
  • Requires accordion to be open for the transition to fire

Trade-offs

  • Pro: Most atmospheric and cinematic feel
  • Pro: Collapsed state still very clean — image is invisible by default
  • Con: 5 full-size background images need loading (performance cost)
  • Con: Background can feel busy with text overlay on some images
  • Con: View Transition source requires expand

Mobile

Background peek still works. Framed image stacks below text. Reduce background opacity to 10% on mobile for readability.

Split Expand

Clean header, but the expanded body splits into a 55/45 layout. The right 45% reveals the image with a clip-path animation — like a curtain drawing open.

Pattern Notes

How It Works

Collapsed state is pure text (identical to current). On expand, the body uses a 55%/45% split. Left side has description + tags. Right side reveals the image via clip-path: inset(0 100% 0 0) → inset(0) — a horizontal curtain reveal. A thin vertical border separates the two halves.

View Transition Wiring

  • The split image gets transition:name — it has a clear rectangular bounding box
  • The 45% width at 1200px container = ~540px wide, 16:10 ratio = solid transition source
  • The clip-path animation and the View Transition can coexist — clip-path completes before navigation

Trade-offs

  • Pro: Clean collapsed state, dramatic reveal
  • Pro: Large image gives strong visual preview and solid View Transition source
  • Pro: The curtain reveal feels premium and intentional
  • Con: Expanded state takes more vertical space due to the split
  • Con: Requires expand for View Transition

Mobile

Stacks to single column. Image appears above or below text at full width. Clip-path reveal still works vertically.

Image Column

A dedicated small image column sits at the far right of each row (always visible, 80px). On expand, a larger 320px version fades in within the body — the small acts as a preview, the large as the transition source.

Pattern Notes

How It Works

Similar to Variation 2 (Persistent Thumbnail) but the image is positioned at the far right, after the tagline — feels like a dedicated "preview column." On expand, the body shows a larger 320px image alongside description + tags. The small column thumb and the large image share the same source.

View Transition Wiring

  • The row thumbnail (always visible) gets transition:name — works without expanding
  • Alternatively, the expanded larger image can be the source for a bigger, smoother morph
  • Consider: use the always-visible thumb as source, and on the detail page, match it to the hero

Trade-offs

  • Pro: Always-visible transition source (best for View Transitions reliability)
  • Pro: Clear visual hierarchy — images have their own "lane"
  • Con: Two images (thumb + expanded) can feel redundant
  • Con: Row is wider/busier than the pure typographic version

Mobile

Column thumbnail shrinks. Expanded image goes full-width below text.

Offset Editorial

On expand, the image floats outside the content boundary — offset to the right with a slight rotation and dramatic shadow. Magazine editorial feel — the image breaks the grid intentionally.

Pattern Notes

How It Works

Collapsed state is pure text. On expand, the description sits in the left ~55%. The image is absolutely positioned to the right, offset beyond the container edge by -1.5rem, and shifted up by -2rem to overlap the header slightly. A -1deg rotation and heavy drop shadow give it a "placed on desk" editorial feel.

View Transition Wiring

  • The offset image gets transition:name — it's an <img> with clear bounds
  • The rotation and offset position mean the browser will animate FROM this tilted position TO the full-bleed hero — dramatic effect
  • Use transition:animate="morph" in Astro for smooth size + position + rotation interpolation

Trade-offs

  • Pro: Most visually distinctive — editorial, magazine-like
  • Pro: The offset breaks monotony in the list
  • Pro: View Transition from rotated → straight creates a satisfying morph
  • Con: Can overlap adjacent items if multiple are open
  • Con: Requires careful z-index management
  • Con: Offset may clip on smaller viewports

Mobile

Falls back to inline (no offset/rotation). Image goes full-width below text. The editorial effect is desktop-only.
